# ja-i18n Skill — Futari 日文翻譯維護

此 skill 涵蓋兩個工作：

1. **審查 / 翻譯**：找出 `ja.ts` 中真正未翻譯的 key 並補上日文
2. **白名單維護**：更新合法漢字白名單，避免把正確日文誤判為繁體中文殘留

---

## 重要背景

Futari 的 `ja.ts` 大量使用**漢字**，這是正確的日文書寫——不是未翻譯的繁體中文。
審查前**必須**先載入白名單，否則會產生大量假陽性。

白名單位置：`references/kanji-whitelist.md`（本 skill 目錄下）

---

## 工作流程 A：審查並修正未翻譯 key

### 1. 載入白名單

先讀 `references/kanji-whitelist.md`，取得白名單詞彙清單。

### 2. 掃描可疑 key

```bash
# 找出疑似繁體中文殘留的 key（排除白名單後）
# 策略：找含有常見繁體中文字型的 value
# 例如：帳、號、設、體、資、際、費、費、對、圖、國、電...
# 但白名單中的詞（食費、光熱費、設定等）要排除
```

判斷原則：
- value 中出現**白名單以外**的繁體中文字（如：帳本、收據、篩選、幣別）→ 可疑
- value 中只有白名單詞彙（食費、設定、確認 等）→ 跳過
- value 為純假名或純英文 → 跳過

### 3. 逐一判斷可疑 key

對每個可疑 key：
1. 對照 `zh-TW.ts` 的對應值，判斷是否是繁體中文直接複製
2. 若 value 完全等同 zh-TW，幾乎確定是未翻譯
3. 若 value 包含漢字但有日文接尾助詞（の、な、を、が、で、に...）→ 可能是正確日文，需再確認

### 4. 翻譯確認未翻譯的 key

翻譯規則：
- **分類名**：優先用漢字（食費、住宅、医療...）
- **UI 動詞**：優先用漢字（設定、確認、保存...）
- **Dialog 否定按鈕**：用「キャンセル」（非「取消」）
- **描述性文字**：自然日文，不要機翻腔
- **語序**：SOV（主語・目的語・動詞），助詞不省略

語尾助詞規則（App 情境）：
- 按鈕文字：無語尾（「保存」「削除」而非「保存する」）
- 說明文字：丁寧體（～ます、～てください）
- 空狀態文字：丁寧體或無語尾均可，保持一致

### 5. 開 PR

- Branch 命名：`fix/ja-i18n-untranslated-<hash>`
- PR 標題：`fix: ja i18n — 未翻譯 key 補完`
- PR body 列出修正的 key 數量和範圍

## 工作流程 B：更新漢字白名單

當遇到新的有爭議漢字（審查時不確定是否為合法日文），執行以下流程：

### 1. 網路搜尋確認

搜尋關鍵字組合（依詞彙選擇）：
- `「[詞彙]」 家計簿 site:zaim.net`
- `「[詞彙]」 夫婦 家計管理`
- `「[詞彙]」 マネーフォワード OR Zaim OR OsidOri`
- `「[詞彙]」 家計簿アプリ UI`

目標：確認該詞是否在日本家計簿 App 的實際 UI 或說明文章中以漢字形式出現。

### 2. 判斷標準

| 情境 | 判斷 |
|------|------|
| 在 Zaim / マネーフォワード / OsidOri 的 UI 截圖或文章中看到該漢字 | ✅ 加入白名單 |
| 只在中文頁面或台灣 App 出現 | ❌ 不加入，需翻譯 |
| 有同音異義問題 | ⚠️ 加入白名單但附上邊界案例說明 |
| 漢字與假名並存（如「娯楽」vs「エンタメ」） | ✅ 若 Futari 已選定漢字寫法，加入白名單並記錄 |

### 3. 更新白名單文件

編輯 `references/kanji-whitelist.md`：
1. 將新詞加入對應的分類表格（支出分類名 / 財務術語 / UI 動詞）
2. 若有邊界案例，加入「需注意的邊界案例」區段
3. 更新頂部「最後更新」日期

### 4. Commit 白名單更新

```bash
git add .claude/skills/ja-i18n/references/kanji-whitelist.md
git commit -m "docs: 更新 ja 漢字白名單 — 新增 [詞彙]"
```

白名單更新可以和翻譯修正一起放在同一個 PR，也可以獨立。

## 白名單快速參照

完整白名單在 `references/kanji-whitelist.md`，以下是核心詞彙：

**支出分類**：食費・光熱費・通信費・交際費・日用品・住宅・医療・教育・娯楽・交通・保険・旅行・車

**財務術語**：収入・支出・予算・残高・合計・資産・精算・決済

**UI 動詞**：設定・更新・記録・確認・追加・保存・編集・削除

⚠️ **邊界案例**：Dialog 否定按鈕用「キャンセル」，不用「取消」；「精算」(費用結算) ≠「清算」(公司廢業)
