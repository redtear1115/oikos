# 記帳 / 家庭財務 App — 用戶反饋分析

> 分析日期：2026-05-09  
> 資料來源：App Store、Google Play、Reddit、PTT、Dcard、Trustpilot、NerdWallet、CNBC Select 等  
> 目標：從競品用戶評價萃取「用戶真正在乎什麼」，對應 Futari 的設計決策

---

## 一、各競品用戶評價摘要

### Honeydue

**⚠️ 重要現況：** 2024 年 Mission Lane 將 Honeydue 剝離成 Moneydue Inc.，目前僅剩 1–10 人維護。停卡計畫（debit card program）已於 2023 年 9 月終止，最後一版更新為 2024/07。支援信箱近月完全無回應，用戶開始出現「這是下架前兆？」的討論（前車之鑑：Honeyfi / Firstly 停服前也類似）。

**用戶稱讚（positive）：**
- 完全免費、沒有 paywall，是最大吸引力
- 夫妻可以選擇哪些帳戶共享、哪些私密，靈活度高
- 共同帳戶一眼看到雙方消費，「終於可以討論錢了」
- in-app 聊天功能讓針對特定支出的對話變容易

**用戶抱怨（negative）：**
- 客服完全失聯，多封信沒有回音，甚至連自動回覆都沒有
- 銀行同步常出問題：餘額更新了，但 transaction 要兩天後才進來
- App 每隔幾天就被踢出，重新登入流程很麻煩
- 功能停滯多年，「rollover budgeting」等常見需求多年沒有回應
- 只有 mobile，沒有 web 版，電腦族不友好
- 廣告干擾體驗（免費模式的代價）

**Feature requests：**
- 跨月預算 rollover
- 資產追蹤（投資、退休金）
- Web / desktop 版本
- 支出分析報告（月比較、年比較）

---

### YNAB

**用戶稱讚（positive）：**
- 方法論本身（zero-based budgeting）改變了用戶的消費行為，平均第一年省 $6,000
- 「終於清楚自己的錢去哪了」是最常見的讚美
- 每天 Zoom workshop、教育資源豐富，學習曲線陡但有人扶
- 無廣告、不賣用戶資料，信任感強
- 最多 6 人共用一個訂閱，CP 值對家庭用戶高
- 2025 Smart Money Awards 年度最佳記帳 App

**用戶抱怨（negative）：**
- 學習曲線太陡，一般人要 2–4 個月才上手；信用卡的處理邏輯尤其令人困惑
- 價格逐年調漲，$14.99/月對只想簡單記帳的用戶太貴
- 銀行連結在美國以外支援差（台灣完全不適用）
- 不追蹤投資帳戶，財富管理用戶不滿足
- 強迫要主動管理每一分錢，沒時間的人堅持不了
- 特定銀行（Capital One、Rogers Bank 等）連結長期有 bug

**Feature requests：**
- 投資 / 退休帳戶追蹤
- 更簡單的入門模式（不強迫 zero-based）
- 更好的行動優先體驗（目前主力還是偏 web）

---

### Spendee

**用戶稱讚（positive）：**
- 介面設計好看，圓餅圖 / 支出分析視覺清楚
- 多幣種支援強，旅行族喜愛
- AI 收據掃描方便
- Shared wallet：只有 owner 要付費，邀請另一半免費加入

**用戶抱怨（negative）：**
- 銀行同步突然停止，「整個 app 變得沒用」
- 曾發生**用戶多年資料被刪除**，補償只有 3 個月免費訂閱，信任崩塌
- 新版 UI 改版讓「月預算剩餘金額」需要多點一層，反彈很大
- App 載入慢，有時輸入一筆花了 30 秒
- 沒有 2FA，資安感知差
- 共享 wallet 無法連結銀行帳戶（只能手動記帳）
- 客服回應慢

**Feature requests：**
- 共享 wallet 支援銀行同步
- 多個 budget（免費版只有一個）
- 跨月預算 rollover
- 2FA 安全機制

---

### Money Lover

**用戶稱讚（positive）：**
- 終身買斷選項，「花一次就是我的」，台灣 / 東南亞用戶偏好
- 多錢包設計靈活（目標錢包、借貸錢包）
- 可匯出到 Google Sheets，資料掌控感強
- 長期用戶（5 年以上）忠誠度高

**用戶抱怨（negative）：**
- 日期選擇 bug：選 21 號會存成 20 號
- 銀行帳戶更新不即時，不是 on-demand 而是定時批次
- 免費版只有一個錢包，限制太大
- 免費版有廣告
- 報表功能相對弱
- 完全沒有雙人 / 家庭協作功能

**Feature requests：**
- 雙人共同記帳
- 更豐富的報表分析
- 負債追蹤（欠款但還未扣款的場景）

---

### 記帳城市（Fortune City）

**用戶稱讚（positive）：**
- 遊戲化設計讓「不愛記帳的人開始記帳」，這是最大差異化
- 城市建築視覺反饋有成就感
- 訂閱價格便宜（約 NT$80/月），入門門檻低
- Google Play 年度精選加持，口碑傳播強

**用戶抱怨（negative）：**
- 城市變大後 app 變慢、耗電
- 遊戲廣告彈出體驗差，全螢幕廣告令人沮喪
- 功能太精簡，沒有轉帳功能、進階財務分析弱
- 幾乎所有功能要自己摸索，說明文件少
- 城市越大越雜亂，「市民求助板」圖示不清楚
- 完全沒有雙人協作功能

**Feature requests：**
- 雙人 / 家庭共同記帳
- 進階財務分析（月比較、年比較）
- 更多遊戲互動元素
- 建築分類聚合（不用一個一個點）

---

### CWMoney

**用戶稱讚（positive）：**
- 台灣電子發票整合是殺手級功能，「載具綁定後自動記帳」體驗極佳
- 免費無廣告，信任感高
- 52 個月 Google Play 台灣精選，長期穩定
- 介面簡潔，新手友好
- 500 萬用戶口碑

**用戶抱怨（negative）：**
- 訂閱制而非買斷，台灣用戶普遍排斥訂閱；有 bug 卻遲遲未修
- 新版本查詢功能退化：以前能按帳戶 + 分類篩選，現在只能搜尋備註和時間
- 子分類篩選無法使用，不同幣種無法統一換算排序
- 共享帳本（最多 3 人）是附加功能，雙人場景體驗設計薄弱
- 年度收支統計要訂閱才能看，基礎需求鎖在 paywall 後面

**Feature requests：**
- 買斷選項（台灣用戶強烈偏好）
- 更強的查詢 / 篩選功能
- 更完整的雙人協作
- 資產追蹤（保險、車輛等）

## 二、跨競品共同痛點彙整

### 🔴 最常被抱怨的問題（頻率最高）

**1. 銀行同步不穩定**
幾乎所有支援 bank linking 的 app 都有這個問題。Honeydue、Spendee、YNAB 的用戶都大量抱怨——餘額 OK 但 transaction 延遲、特定銀行長期連不上、同步突然整個斷掉。這是整個 fintech category 的基礎設施問題，不是個別 app 設計問題。

→ **Futari 的處理方式**：不做 bank linking（手動記帳），反而完全迴避了這個最高頻痛點。手動記帳是「必須告別的舊做法」的常見抱怨，但實際上 bank sync 的痛苦也不小，且台灣銀行 API 開放程度有限。

**2. 客服失聯 / 開發停滯**
Honeydue 現況（1–10 人維護、客服無回應）讓老用戶高度焦慮。Spendee 刪用戶資料事件雖非開發停滯，但讓信任崩塌。用戶對「app 會不會哪天消失」的焦慮是真實的。

**3. 訂閱制 vs 買斷制的文化摩擦（台灣特有）**
PTT / Dcard 反覆出現「可以接受買斷，不接受訂閱」的聲音。CWMoney 的訂閱制被台灣用戶批評，MOZE 的訂閱同樣有類似聲音。這是台灣市場的特殊文化，跟西方用戶接受度不同。

**4. UI 改版破壞既有習慣**
Spendee 新版把月預算剩餘藏深了一層，反彈激烈。CWMoney 新版查詢功能退化。用戶對「改壞了」的反應比「加新功能」更強烈。

**5. 功能鎖在 paywall 後的不滿**
基礎功能（年度統計、共享帳本、進階分析）一旦被放進訂閱才能用，用戶感知是「被剪掉的免費版」而非「升級版」。

## 三、三維分析

### 必備功能（沒有就流失）

這些功能在 1–2 星評論裡最常被提到為流失原因：

**快速記帳**：每次記一筆的摩擦必須極低。用戶流失最快的原因之一是「記帳太麻煩，放棄了」。PTT 上明確提到「每次記帳動作越簡單越好」、「不用開 app 就能快速輸入」。

**分類系統**：預設分類要夠覆蓋日常，但也要能自訂。沒有自訂分類是重複抱怨點。

**基本報表**：月支出趨勢、分類佔比（圓餅圖）。這是最低門檻，幾乎所有 app 都有，沒有就直接出局。

**資料安全感**：Spendee 刪資料事件顯示，資料遺失是致命的。用戶願意忍受 bug，但無法接受資料消失。雲端備份或本地備份是基礎期待。

**穩定性**：App 頻繁 crash、被踢出登入、同步卡死，是 1 星評論最常見原因。功能再好，不穩定就白搭。

---

### 差異化功能（有了就好評、形成口碑）

**遊戲化機制（記帳城市）**：「讓原本不記帳的人開始記帳」是能被口碑傳播的功能。雖然長期留存未知，但獲客效果真實。

**台灣電子發票整合（CWMoney）**：每次在超商消費不需手動輸入，「綁載具自動記帳」大幅降低記帳摩擦。台灣市場特有的殺手功能。

**方法論框架（YNAB）**：用戶不只是在用 app，而是在「學習一套理財系統」。高度認同感讓用戶在不便宜的訂閱上維持高留存。

**共享帳本的即時性**：Honeydue 用戶稱讚的是「終於可以一起看」，Spendee 用戶稱讚 shared wallet 的即時反映。雙人 app 的核心期待是「同步感」，不只是「看得到」。

**買斷選項（台灣市場）**：Money Lover 的終身買斷在台灣市場特別有吸引力，MOZE 在台灣的接受度部分也來自長期投資感。

---

### 最常被抱怨的設計問題

**UI 資訊層級太深**：把重要數字（月預算剩餘、當月餘額）藏進去要多點幾層，是最容易引發憤怒的設計決策。用戶的期待是「打開就看到最重要的數字」。

**強迫複雜度**：YNAB 被批評的核心是「必須理解它的方法論才能用」。希望簡單的用戶感覺被強迫學習。→ 反面：Futari 的「極簡」定位正好對位這個痛點。

**廣告破壞體驗**：記帳城市的全螢幕廣告彈出被強烈抱怨。免費 app 靠廣告的模式在體驗上有天花板。

**資料無法帶走**：用戶換 app 最大障礙是「多年的資料怎麼辦」。CSV 匯出、跨 app 匯入需求是高頻 feature request，PTT 上有多篇討論。

**沒有 Web 版**：Honeydue 缺 web 版被多次提到，特別是習慣在電腦做財務管理的用戶。不過台灣手機優先文化下這個需求稍弱。

**隱私 / 資安感知不足**：Spendee 沒有 2FA 被用戶批評。財務 app 的資安期待比一般 app 高，但很多 app 沒有配套。

## 五、給 Futari 的觀察

**台灣電子發票整合是值得認真評估的優先項**。用戶在 PTT 上明確說「需要載具自動匯入記帳，因為我在超商消費很多」。CWMoney 靠這個建立了護城河，但它沒有雙人場景。Futari 若能把「載具整合 + 雙人共同記帳」結合，在台灣市場幾乎沒有競品。（注意：cloud-invoice-design.md 裡已有此規劃，卡在 APP_ID 申請。）

**定期收支的「儀式感」是差異化敘事**。競品的定期功能都是「自動扣帳」，Futari 的 preview→commit 模型在功能上看起來多一步，但在敘事上是「薪水入帳是兩個人一起確認的時刻」。這個敘事如果在 onboarding 或 App Store 文案上說清楚，是吸引目標用戶的強力訊號。

**Honeydue 的衰退是時間窗口**。現況是 1–10 人維護、客服失聯、用戶焦慮。這批用戶正在尋找替代品，且他們已經被「雙人記帳」的價值說服過，只需要一個更可靠的選項。

**「資料會不會消失」是用戶底層焦慮**。Spendee 刪資料事件的陰影很大。Futari 的 Supabase 架構 + 加密設計是可以主動宣傳的信任點，但目前產品層沒有把這個說出來。
