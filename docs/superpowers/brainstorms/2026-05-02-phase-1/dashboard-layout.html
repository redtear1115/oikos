A · FAB（浮動按鈕）

右下角懸浮「+」。最常見的 Material pattern，記帳類 app 標配（Splitwise / Tricount）。優點：list 可佔滿畫面；缺點：拇指要伸到角落。

B · 底部固定 bar

整條按鈕固定在最下方。優點：拇指輕鬆按、文字可說明意圖；缺點：永遠占走 ~60px 高度。適合「新增」是壓倒性主要動作的場景。

C · Inline quick-add（永遠展開）

表單常駐在 list 上方。優點：0 tap 開始輸入，最快；缺點：占畫面、複雜欄位（描述、category）放不下，要退到全頁。適合「金額 + 分攤」就 90% 夠用的場景。

推薦

A · FAB。你的 case 是 60 秒內第一筆紀錄，需要描述 + 分攤 + category 才能算清楚誰欠誰，所以表單需要全頁展開。FAB 不犧牲 list 視覺、又是 mobile 用戶最熟悉的 mental model（Splitwise / Tricount / 麻煩記帳都這樣）。
